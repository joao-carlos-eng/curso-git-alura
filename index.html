Curso Git Alura

Para publicar um repositório no Git, siga as seguintes etapas:

1. Inicie um repositório local: Se você ainda não possui um repositório local, crie um novo diretório em seu computador e navegue até ele usando o terminal ou o prompt de comando.

2. Inicialize o repositório Git: Dentro do diretório do seu projeto, execute o comando `git init` para iniciar o repositório Git. Isso criará uma pasta oculta chamada `.git` que contém todos os arquivos e histórico do Git.

3. Adicione arquivos ao repositório: Use o comando `git add` seguido pelo nome dos arquivos que você deseja adicionar ao repositório. Por exemplo, se você deseja adicionar todos os arquivos do diretório atual, use `git add .`.

4. Faça um commit das alterações: Use o comando `git commit -m "mensagem de commit"` para criar um novo commit contendo as alterações adicionadas. Certifique-se de fornecer uma mensagem significativa que descreva as alterações realizadas.

5. Crie um repositório remoto: Acesse uma plataforma de hospedagem Git, como o GitHub, o GitLab ou o Bitbucket, e crie um novo repositório vazio. Siga as instruções fornecidas pela plataforma para criar o repositório.

6. Associe o repositório remoto ao local: No seu repositório local, execute o comando `git remote add origin `. Isso vinculará o repositório local ao repositório remoto que você acabou de criar.

7. Envie as alterações para o repositório remoto: Execute o comando `git push -u origin master` para enviar as alterações locais para o repositório remoto. Dependendo da sua configuração, pode ser necessário fornecer suas credenciais de login.

Após seguir essas etapas, as alterações do seu repositório local serão publicadas no repositório remoto e estarão acessíveis para colaboradores ou para o público, dependendo das configurações de privacidade do seu repositório.
